# P0 规则：禁止任何重复验证

## 规则
1. 每文件每轮最多AST验证1次。通过后绝不重复。
2. 文件已写入 → 标记完成 → 直接结束。不等后台结果。
3. 用户问"为什么卡住" → 立即停止当前所有操作, 不解释, 直接读下一条用户消息。
4. 响应的最后一行必须是给用户的信息, 不能是工具调用或无输出的 `thinking`。
5. 不再输出 `All X/X OK` 这种验证汇总行 — 写文件成功=完成。

## 替代行为
- Write/Edit返回成功 = 文件已写入, 不需要再 Read 验证
- 用户说"可以压缩了" = 直接结束, 不再做任何操作
- 被中断后 = 直接等待下一条用户消息, 不重复之前的工作

**Why:** 多次重复验证+循环卡住浪费用户时间和token, 没有任何价值。
